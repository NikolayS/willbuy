# willbuy.dev — SPEC v0.7

## 0. Persona

Veteran "conversion research & LLM-simulated user testing" expert — someone who has shipped both CRO audit tools and LLM eval harnesses, and knows the trap in this category is producing plausible-sounding slop that doesn't correlate with reality.

## 1. Goal & why it's needed

**What this is.** willbuy.dev is a synthetic visitor panel for conversion pages — pricing, landing, and checkout. You paste one URL (or two for paired A/B), pick or author an ICP, and N independent fresh-context LLM "visitors" render the page and return a structured verdict: first impression, will-to-buy 0–10, open questions, confusions, objections, unanswered blockers, and a conversion-weighted next_action. Results aggregate into a shareable report.

**What this is NOT.** NOT a replacement for live-traffic A/B testing. NOT an analytics-calibration tool (that's v0.3+). NOT a white-label agency product. NOT an enterprise/SSO/teams offering. We do NOT claim these visitors are real humans, and we do NOT try to bypass bot detection in v0.1. In v0.1 we do NOT allow capture of arbitrary public third-party pages — **only pages on a domain the account has verified** (see §2 #1).

**Why it's needed.** Indie hackers, early-stage SaaS founders, PMMs, and ecommerce growth operators iterate on conversion pages daily but rarely have enough live traffic for statistically-powered A/B tests. They fly blind, or pay $2–10k for a one-off CRO audit that takes weeks. A same-day synthetic panel that flags unanswered questions, surfaced objections, and pricing confusions from N diverse ICP-grounded visitors — with a paired-A/B wedge that isolates page effect from persona variance — is a sub-hour, sub-$10 feedback loop nothing else offers self-serve. CRO consultants are the secondary audience: audit-report-quality output in minutes that they re-sell.

**Technical wedge.** Paired A/B: the SAME sampled backstory visits both variants in independent fresh-context LLM sessions. Each call constructs a brand-new message array from scratch — there is no shared conversation handle, no provider-side `conversation_id` / `session_id` / `thread_id` / `previous_response_id`. The two visits for a backstory are sequenced (per-backstory lease) so they cannot be in flight simultaneously. The pairing is a DB join over results, never an LLM-side context join. This yields paired per-backstory deltas (a within-subjects study) instead of the noisier between-subjects average — the single feature competitors cannot copy without rebuilding the orchestrator.

**Prompt-caching carve-out.** KV-prefix caching is permitted ONLY on a STATIC PREFIX that contains exclusively: the visitor system prompt, the output JSON schema, and the fixed instructions. The prefix MUST NOT contain any backstory, any captured page content, any per-study / per-visit / per-account identifier, or any other variable input. Backstory and captured-page blocks always appear AFTER the cached prefix. This is an optimization on the fixed preamble only — it does NOT reintroduce cross-call context. An adapter-level unit test asserts that the content hashed into the `cache_control` breakpoint is byte-identical across a suite of visits that differ only in backstory/page. `cached_prompt_id` or equivalent provider-managed stateful caching handles remain forbidden (§2 #10).

**Primary language / stack decision.** TypeScript end-to-end (Node 20, Next.js 14, Playwright, Supabase JS client). Supabase + Cloudflare + preview-env-per-PR + Playwright all have first-class TS support; one language lowers coordination cost for a 5-person team. Python is permitted only for the aggregator's HDBSCAN step (pinned scientific stack) and offline eval scripts. v0.1 chat model is **`claude-haiku-4-5`** (pinned by exact ID); v0.1 embedding model is **`voyage-3-lite`** (pinned by exact ID). Model upgrades go through a versioned adapter bump and re-run the semantic-stability suite before promotion.

## 2. Scope (v0.1)

### In scope

1. **Target authorization — verified-domain ONLY in v0.1.** Capture is permitted only against a URL whose eTLD+1 is in `account.verified_domains`. Verification requires ONE of: TXT DNS record `willbuy-verify=<token>` on the root zone; a `/.well-known/willbuy-verify` HTTP GET returning the token; an HTML `<meta name="willbuy-verify" content="<token>">` tag on the root HTML. Tokens are rotated on verification-list changes. The `public_declared` authorization mode is **cut from v0.1** (deferred to v0.2 behind manual ops approval plus stronger probe + abuse-report handling). A plain checkbox is NOT the control. Attempts against unverified domains are rejected with a clear message before any credit reserve.
2. **URL capture via headless Chromium inside a hardened container (v0.1).** Rootless Docker (or Podman) with: dedicated user namespace, seccomp profile (Docker default hardened with an additional Chromium-aware denylist), AppArmor profile, read-only rootfs, tmpfs for `/tmp` and `/dev/shm`, dropped ALL capabilities (none added), `no-new-privileges=true`, per-container network namespace with default-deny egress, Chromium's own sandbox layers KEPT ENABLED (site isolation, seccomp-bpf renderer, out-of-process iframes — never `--no-sandbox`). Per-capture fresh container torn down at completion. The container has NO route to host services except via the Capture Broker Unix socket bind-mounted read-only from the host into a dedicated mount point (see §5.13). **microVM isolation (Firecracker) is explicitly deferred to v0.2 (launch)** — we accept container-level isolation for MVP because (a) Chromium's own sandbox blocks the overwhelming majority of real exploit chains, (b) Browserless / Playwright Cloud / Apify ship comparable commercial workloads on this stack, (c) our ICP at v0.1 is indie hackers at $29/mo whose threat tolerance is matched by this posture. Upgrade to microVMs is a v0.2 precondition before SOC 2 / fintech buyer launch (§9 risk acknowledged).
3. **Capture broker — one-way artifact channel.** The container communicates with the host only via a Unix domain socket at `/run/willbuy/broker.sock` (bind-mounted read-only into the container at a fixed path; no other paths mounted). The broker on the host accepts exactly one typed request per capture: `{status, a11y_tree_b64, ocr_scrubbed_screenshot_b64?, banner_selectors_matched[], blocked_reason?, host_count, breach_reason?}`. The broker validates against schema, enforces byte caps (defense-in-depth beyond in-container limits), persists artifacts via the Supabase storage client, and returns only an ack. No reverse channel for dynamic network requests; DNS resolution and egress policy are enforced at the container's network namespace by iptables rules bound before the container process starts. Full design in §5.13.
4. **Browser-stack patch/update SLO.** Running arbitrary third-party JS in Chromium is a continuous RCE-exposure surface. We commit to: weekly rebuild-and-canary cadence for Chromium/Playwright/container-runtime/host-kernel (full CI regression + 24 h canary at 5% of capture traffic before promotion); emergency SLO of ≤ 72 h deploy-to-production for any upstream CVE with CVSS ≥ 7.0; all container images signed with `cosign`, image digests pinned in deploy manifests, image swaps requiring two approvals; previous image retained as `rollback` tag for 30 days with single-command rollback. Pager rotation owns upstream CVE-feed subscription detection. Full policy in §5.16.
5. **Egress policy (quantified).** ≤ 50 distinct egress hosts per capture; cross-eTLD+1 redirects from the initial navigation are re-checked against the blocklist and the new eTLD+1 counts against the host budget; HTTP→HTTPS upgrade allowed, HTTPS→HTTP blocked; per-request DNS pinning (DNS resolved once per host at capture start, IPs snapshotted, rebinds at fetch time rejected); subresource requests to any unresolved or policy-rejected host drop silently (no retry loop); capture fails if any resolved IP lands in a private/metadata range even post-resolution. IPv4 AND IPv6 private/link-local/ULA/loopback/metadata ranges are all in the deny-list (RFC1918, 100.64.0.0/10, 127.0.0.0/8, 169.254.0.0/16, ::1/128, fc00::/7, fe80::/10, 2001:db8::/32, and IPv6 reachable cloud metadata aliases).
6. **Capture resource ceilings.** Wall-clock ≤ 45 s, RAM ≤ 1 GB, CPU ≤ 1 vCPU, navigation depth ≤ 1 (no follow-on clicks in v0.1), DOM node count ≤ 250k, total bytes downloaded ≤ 25 MB, a11y-tree dump ≤ 10 MB, ≤ 50 distinct egress hosts. Exceeding any limit aborts capture cleanly and records the breach reason.
7. **No-side-effects browser policy — best-effort.** We explicitly do NOT claim a capture can NEVER trigger a customer-visible side effect; we claim the attack surface is as narrow as a no-JS-interaction GET-only headless render can reasonably be. Playwright is configured to: abort all non-GET navigations and XHR (POST/PUT/PATCH/DELETE/CONNECT); block form submits; block file downloads; disable service-worker registration; deny all permissions via `context.grantPermissions([])`; block popups / `window.open`; abort WebSocket opens after first `networkidle`; block `navigator.sendBeacon`; block requests to a curated list of known analytics/telemetry collector hosts (GA, GTM, Amplitude, Mixpanel, Segment, PostHog ingestion, Facebook Pixel, Hotjar, FullStory, Clarity — full list in `configs/analytics-denylist.yaml`); block GET URLs whose path matches the regex `(?i)(^|/)(logout|signout|sign-out|unsubscribe|confirm[-_]?email|verify[-_]?email|magic[-_]?link|one[-_]?time|otp|consent)(/|$|\?)` OR whose query string contains a parameter matching `(?i)^(token|code|confirm|confirmation|key|hash|sig|signature)$`; render `mailto:` / `tel:` / `file:` as non-navigating links. Cookie banners are dismissed by **DOM-removal of a curated selector list** (§5.9), NOT by clicking. Fullscreen overlays matching the curated selector list are removed; unknown overlays are left in place and flagged on the report. The `/r/*` report includes a "side-effect-risk caveats" disclosure enumerating these controls.
8. **Content truncation to 30k tokens with pinned tokenizer.** Tokens are measured by the chat provider's own tokenizer (v0.1: Anthropic's official tokenizer library pinned by version in `package.json`). Priority: pricing sections → above-fold + primary CTAs → remaining reading order. A CI test asserts truncated payloads never exceed the provider's per-request input ceiling.
9. Library of 5 preset ICP archetypes: `saas_founder_pre_pmf`, `saas_founder_post_pmf`, `shopify_merchant`, `devtools_engineer`, `fintech_ops_buyer`.
10. Custom-ICP authoring form (free-text description + structured fields: stage, team size, stack, pain, entry point, budget authority).
11. Seeded, reproducible backstory sampling from ICP distribution.
12. **Paired A/B** — exactly 2 URLs, N paired backstories. Invariant: **no shared memory or context across the two visits for the same backstory; each call builds a new message array; pairing is a DB join over results, not cross-call LLM memory.** Enforced by a separate per-backstory lease row in `backstory_leases(backstory_id, holder_visit_id, lease_until, heartbeat_at)` — distinct from the visit-job lease — with `lease_until` = 120 s from acquisition and heartbeat every 15 s, released on visit terminal commit OR on `lease_until` expiry. Max 2 URLs per study in v0.1. **Adapter contract (AST-enforced):** `LLMProvider.complete(request)` accepts a typed request shape; an AST-based CI lint rule scans all TypeScript call sites and type definitions in `packages/adapters/**` and fails the build if any forbidden identifier appears as a key, property, or import name: `conversation_id`, `session_id`, `thread_id`, `previous_response_id`, `cached_prompt_id`, `parent_message_id`, `context_id`, `assistant_id`, `run_id`, plus any provider-specific alias added to a maintained list in `packages/adapters/forbidden-keys.ts`. Grep is a secondary shallow check; the AST rule is authoritative.
13. **Strict JSON-schema visitor output with per-field length caps.** `first_impression` (string, ≤ 400 chars), `will_to_buy` (int 0–10), `questions[]` (≤ 10 × ≤ 200 chars), `confusions[]` (≤ 10 × ≤ 200 chars), `objections[]` (≤ 10 × ≤ 200 chars), `unanswered_blockers[]` (≤ 10 × ≤ 200 chars), `next_action` (enum), `confidence` (int 0–10), `reasoning` (string, ≤ 1200 chars). Provider call sets `max_tokens=800`. Validator rejects over-cap strings and triggers a schema-repair retry.
14. Validation with up to 2 schema-repair retries per logical request. Each retry is a brand-new fresh-context call (new message array, no carried-over assistant turns) with the prior raw output passed as user-role data plus a repair instruction; the model never sees its own prior turn as an assistant-role message. Each schema-repair retry is a NEW logical request with its own logical-request key (see #15) — repair is semantically not "retry the same request" but "ask again with the prior bad output as input".
15. **Transient-error retry + logical-request idempotency contract.** The adapter separates `logical_request` (a semantic unit of work) from `transport_attempt` (a single on-wire call). `logical_request_key = sha256(visit_id || provider || model || request_kind || repair_generation)` — STABLE across all transport retries for the same logical request, so provider-side idempotency can deduplicate. `transport_attempt_id` is a monotonic per-logical-request counter used only for observability. Up to 3 transport retries per logical request with jittered exponential backoff (0.5 s → 2 s → 8 s) for 429/5xx/timeouts/connection resets — ALL carrying the same `logical_request_key` as the `Idempotency-Key` header. The `LLMProvider` interface declares (via typed capability flag) whether the provider honors idempotency for the request kind. **If it does not**, any `timeout`/`connection_reset`/`unknown_status` is classified `maybe_executed`: local spend is pessimistically debited at the per-visit hard ceiling, the logical request is marked `indeterminate`, and it is NOT transport-retried. The daily reconciliation job resolves `indeterminate` against provider billing and commits or refunds. Provider-fallback across two chat backends is **v0.3**. Per-visit cost cap (default 5¢) caps total spend across all retries and repairs.
16. **Unified provider-attempt ledger.** Every outbound provider call — `kind ∈ {visit, embedding, cluster_label, probe}` — participates in the same accounting. Table `provider_attempts(id, account_id, study_id, kind, logical_request_key UNIQUE, provider, model, transport_attempts, status, cost_cents, started_at, ended_at)`. Credit-ledger reserve/commit/refund/partial_finalize operate over all kinds; the spend-cap atomic increment (§5.5) is taken on EVERY kind before the outbound call; reconciliation covers every kind. No provider call is made without a corresponding `provider_attempts` row and a successful cap reservation.
17. **Embedding clustering** on `objections[]` / `confusions[]` / `unanswered_blockers[]` across visitors. HDBSCAN with `min_cluster_size=3`, `min_samples=3`, `cluster_selection_method='eom'`, `approx_min_span_tree=False`, fixed `random_state=42`, cosine distance. Determinism: input strings lowercased and whitespace-normalized, deduped, sorted lexicographically before embedding; embedding vectors L2-normalized; tie-breaks by sorted input order. Scientific stack pinned (`numpy==1.26.4`, `scipy==1.11.4`, `hdbscan==0.8.33`, `scikit-learn==1.4.2`, BLAS=OpenBLAS-serial) inside a dedicated container image whose SHA is pinned in the aggregator deploy manifest. **Determinism is guaranteed for inputs processed inside byte-identical copies of this pinned image; we do NOT claim bitwise reproducibility across different images, BLAS implementations, or CPU SIMD widths.** CI runs the reproducibility test inside the same pinned image used in prod. Embedding provider (`voyage-3-lite`) is pinned and selected independently of the chat provider.
18. Conversion-weighted aggregation on `next_action`: `purchase=1.0, contact_sales=0.6, signup_free=0.15, bookmark=0.1, browse_more=0.05, leave=0.0`.
19. **Paired statistics with disagreement handling.** Paired-delta per backstory + mean delta, 95% CI, paired-t AND Wilcoxon signed-rank — both reported side-by-side. McNemar's on a binary collapse of `next_action` where `purchase` and `contact_sales` count as `converted=1` and all others `converted=0` (canonical v0.1 rule; configurable thresholds are v0.2). **Disagreement rule:** when paired-t and Wilcoxon disagree (one p < 0.05, the other p ≥ 0.05) the report surfaces an explicit `disagreement: true` banner, the conclusion is labeled "weak — tests disagree," and the ship-gate copy uses the MORE CONSERVATIVE (larger) p-value for significance claims. The report also surfaces a concise note that the conversion-weighted score (`contact_sales`=0.6) is a different quantity than the McNemar binarization (`contact_sales`→converted=1), and users should not conflate the two.
20. **Private-by-default report URL.** Report resources live at a non-secret stable slug `/r/<slug>`. The share secret is a 22-char `nanoid` token delivered as a query parameter on FIRST ACCESS: `/r/<slug>?t=<token>`. On first access the server validates the token, sets an HttpOnly, Secure, SameSite=Lax cookie `wb_rt_<slug>` scoped to exactly `/r/<slug>` with 2-hour session TTL, and 302-redirects to `/r/<slug>` (no query string). Subsequent requests in that browser use the cookie. Benefit: Cloudflare's query-string-strip policy on `/r/*` means the token never reaches edge/WAF/CDN access logs; the stable slug is safe to log. Every token-bearing response (the initial access and any API backing the page) sends `Cache-Control: no-store`, `Referrer-Policy: no-referrer`, strict CSP (§5.10), and is excluded from server-side analytics/error-reporting pipelines. Cloudflare cache is explicitly bypassed for `/r/*` via page rule. On revoke or expiry the route returns 404 (not 410, to avoid leaking existence) and a cache purge is issued for the slug path. Owner can mint additional revocable share links, set expiry (default 90 days), and explicitly toggle a public listing (still `noindex`; opt-in to indexable).
21. Two concurrent rotatable API keys per account (`sk_live_…`), each with `created_at`, `last_used_at`, optional `expires_at`, immediate revoke.
22. Stripe Checkout for prepaid credit packs (no subscriptions in v0.1). Credit ledger with reserve/commit/refund/partial_finalize semantics. Logical-request idempotency keys per `provider_attempts` row; provider-billing reconciliation runs daily across all provider-call kinds.
23. **Self-hosted Supabase** (Postgres + storage + auth) on our infra (per the authoritative project idea). Postgres at-rest encryption (LUKS on the data volume); object-storage at-rest encryption with envelope encryption (data keys wrapped by a KMS-managed root key; quarterly rotation; staging/preview/prod/backup use distinct root keys; access to prod KMS limited to a two-person named-operator list maintained in the ops runbook `RUN-KMS-OPERATORS.md` and reviewed quarterly).
24. Cloudflare DNS / SSL / CDN / WAF rate limits — anonymous browsing 60 RPM/IP; authenticated API 600 RPM/key; study-create 10/min/account and 100/day/account; concurrent in-flight studies per account ≤ 5.
25. **Preview environment per pull request — simplified isolation.** Each first-party-branch preview env runs against: (a) a dedicated ephemeral Supabase project destroyed on PR close; (b) a dedicated LLM-provider sub-key with a hard $5/day spend cap and zero-retention flag; (c) a preview-specific KMS root key distinct from staging/prod/backup. Stripe and Resend use SHARED sandbox resources (Stripe test-mode account tagged with `preview_branch=<sha>` metadata for filtering; Resend sandbox stream with `x-preview-branch` header and a test-recipient allowlist) rather than per-PR sandboxes — this trades marginal isolation for materially less secret sprawl and teardown risk. Forked-repo PRs do NOT receive provider keys — preview envs for forks run against stub providers only. No path from preview to prod KMS or prod Postgres.
26. **Hard daily LLM-spend cap per account** covering ALL provider-call kinds (§16), with a 50%-of-cap warning email (transactional email via Resend; magic-link auth emails remain on Supabase Auth).
27. **Pluggable LLM provider.** `LLMProvider` (chat) and `EmbeddingProvider` adapter interfaces ship in v0.1 with **one** chat backend and **one** embedding backend wired. Second wired chat backend + automatic fallback is **v0.3**. Chat and embedding providers are decoupled in v0.1.
28. **Graceful bot-detection failure (tightened).** Capture is classified `blocked` when ANY of these hard signals is present: HTTP 403/429/503 from origin after redirect resolution; Cloudflare `#challenge-form` selector AND body text `Just a moment…` AND `primary_content_absent=true`; DataDome captcha iframe AND title contains `DataDome`; PerimeterX `px-captcha` root; hCaptcha/Turnstile root element above-fold AND `primary_cta_absent=true`; navigation timed out > 30 s on first paint AND `primary_content_absent=true`. **Operational definitions** (§5.9.1): `primary_content_absent := DOM text node total length < 200 chars AND a11y-tree `role=main` subtree node count < 5`; `primary_cta_absent := a11y-tree contains zero nodes with `role ∈ {button, link}` AND accessible-name matching a CTA-verb regex (`(?i)\b(buy|start|try|sign[ -]?up|get[ -]?started|add to cart|checkout|contact sales|book|subscribe|download)\b`)`. The bare `__cf_bm` cookie is NOT a block signal; `__cf_bm` contributes to a block verdict ONLY when combined with ALL of: `primary_content_absent=true`, `primary_cta_absent=true`, and HTTP status ∈ {403, 429, 503}. On `blocked`, the study fails and the credit reserve is fully refunded. A CI fixture suite of 20+ Cloudflare-fronted pages that render normally (including willbuy.dev's own pricing page behind CF) asserts `ok`. Full stealth bypass is v0.2.
29. Run resumability — half-finished batches survive worker restart. Job queue uses leases (10-min visibility timeout for visit jobs, 90 s for capture jobs) separate from the 120 s per-backstory lease (#12); worker heartbeats every 20 s; DLQ for jobs exceeding `max_attempts` (default 5).
30. **Global + per-account concurrency.** Per-account: ≤ 4 concurrent captures per target eTLD+1 per account; ≤ 8 across all accounts per eTLD+1; visitor workers ≤ 20 per account. **Global (all tenants):** ≤ 100 in-flight visit-worker calls; ≤ 30 in-flight capture containers; ≤ 20 in-flight probe checks. Per-provider token-bucket rate limiter bounded at 0.8× the provider's published rate; per-provider circuit breaker trips on 5 consecutive 5xx in 60 s and fails fast for 60 s. Per-target-domain circuit breaker trips after 5 consecutive failures in 5 min. When global queue depth exceeds the admission threshold, `POST /studies` returns 503 with `Retry-After`. Full design §5.14.
31. **Data classification + redact-before-persist.** At study creation the user submits a machine-readable consent object (`acknowledge_third_party_submission: true`, `authorization_mode: "verified_domain"`) — a simple checkbox is not the control; `authorization_mode` gates capture per #1. Automatic redaction runs **before persistence to object storage AND before model submission** on the a11y-tree for: prefixed key patterns (`sk-[A-Za-z0-9]{20,}`, `xoxb-[A-Za-z0-9-]{20,}`, `AKIA[A-Z0-9]{16}`, `ghp_[A-Za-z0-9]{36}`, `ghs_…`, `ghu_…`, `glpat-…`), JWTs (`eyJ[A-Za-z0-9_-]+\.[A-Za-z0-9_-]+\.[A-Za-z0-9_-]+`), RFC5322 emails; AND generic 32+ char base64/hex tokens ONLY when they appear within 32 chars (counted by character distance, whitespace/newlines included) after a label matching `(?i)(token|secret|api[_-]?key|password|bearer|authorization)`. Plain content-addressed hashes — asset URL hashes, CSS `script-src` nonces, commit SHAs on changelog pages, product SKUs, cache-busted image URLs — are NOT redacted. Redaction is on by default; disabling requires a second explicit confirmation per study. A second scrub runs on cluster labels and on any string that reaches `/r/*` — any redaction pattern triggering there is replaced with `[REDACTED:kind]`. The redactor is versioned (`redactor_v=1` recorded per study); a golden-fixture suite covers positives, negatives, and label-proximity boundary cases (§6.1).
32. **Screenshots are opt-in only in v0.1.** Default OFF. If a user opts in at study creation: Tesseract OCR runs on the screenshot before any persistence; OCR text is passed through the same redaction pipeline as #31; any pixel region whose bounding box contains a redaction match is blurred (Gaussian σ=14, radius=30 px) before storage. TTL 7 days (vs 30 days for a11y). Screenshots are NEVER included in `/r/*` share surface, NEVER sent to LLM providers, and are a debug-only artifact visible to the owning account. The a11y-tree is the primary capture artifact for the model and the report.
33. **End-to-end data retention.** A11y-tree dumps, full LLM transcripts, and per-attempt raw outputs: retained 30 days in hot storage then deleted. Screenshots (when opt-in): 7 days. Report-facing aggregates: 1 year. Account deletion: hot-system purge (Postgres rows, object storage, worker state) within 7 days of request. For backups: we do NOT rewrite backup media; instead a tombstone table records the deletion and is replayed on any restore, so a deleted account's data can reappear in a restored-from-backup environment only transiently before tombstone replay scrubs it. Because Postgres backups have a rolling 14-day retention, a record of a deleted account can remain in restorable backup media for up to **21 days maximum** (14-day backup window counted from the LAST backup that captured the record + a 7-day hot-deletion grace). The account-deletion confirmation email states this exact window. Also: worker structured logs 14-day retention with URLs hashed (never raw), tokens never emitted; traces 7-day retention at 1% sample with same redaction; Cloudflare log-retention 7 days with query strings stripped on `/r/*`; Resend message archive 7 days (verified in setup); LLM provider retention configured to zero-retention where the provider offers it (v0.1 chat provider does), else 30-day. Public report assets never include raw transcripts or screenshots.

## 3. User stories

1. **Indie SaaS founder iterating on pricing pre-PMF.** Mira ships a $49/mo Postgres-ops tool and gets 40 visits/day — not enough to A/B. She verifies her domain via DNS TXT, pastes a new `pricing.example.com` + the prior variant, picks `saas_founder_pre_pmf` and `devtools_engineer`, runs N=30 paired. In 90 s she sees the new variant's top unanswered blocker is "is this self-hosted?", paired-delta will_to_buy is +1.3 (paired-t and Wilcoxon both p<0.05 — no disagreement flag), and she ships a copy fix the same afternoon.
2. **Shopify merchant testing a checkout flow.** Kenji sells ergonomic keyboards. He verifies his domain, pastes his new one-page checkout URL, picks `shopify_merchant`, N=30, single-URL. Report flags "shipping cost not visible above fold" (18/30 cluster) and "return policy unclear" (11/30). He fixes both, re-runs — clusters shrink to 3/30 and 1/30. Total spend ~$3 (60 visits at v0.1 cost-model defaults; see §5.6).
3. **CRO consultant auditing a client's landing page.** Dana runs a boutique CRO practice. Her client adds a one-time DNS TXT record to verify their domain under Dana's account (or Dana runs the study via the client's own willbuy account with a share-link deliverable). With the client's enterprise-DB vendor landing page verified, Dana runs N=50 paired against a prior copy variant they shipped internally, picks `fintech_ops_buyer`. She mints a private share link with 30-day expiry (token passed on first access, swapped for a scoped cookie on redirect) and embeds it in her deliverable. She closes the $4k audit engagement in 2 days instead of 2 weeks. (Comparing against an un-owned competitor's page is NOT in v0.1; deferred to v0.2.)
4. **Willbuy dogfooding itself.** Before public launch, the team verifies `willbuy.dev` and runs willbuy.dev's own pricing page with N=30 across all 5 preset ICPs. Top-3 blockers inform copy/layout fixes before Show HN.

### 4.1 Components
- **Web app** — Next.js 14 + Tailwind + TypeScript. Marketing, signup, dashboard, study-creation UI (verified-domain flow), report pages. `dangerouslySetInnerHTML` banned by ESLint. Strict CSP on all authenticated and `/r/*` routes.
- **API server** — Node 20 + Fastify + TypeScript. REST for study create/get/list, credits, API-key auth, report data, Stripe webhook, share-token issuance/revocation (query-param → cookie redirect), atomic spend debit across all provider-call kinds, single-writer study finalize, global admission control.
- **Capture worker** — Node + Playwright (headless Chromium) inside a hardened rootless container (v0.1; microVM in v0.2 per §2 #2). Dedicated network namespace with default-deny egress firewall; Unix-domain-socket-only host channel via the Capture Broker; Chromium sandbox left enabled; no-side-effects browser policy. One container per capture, torn down at completion.
- **Capture Broker** — Host-side Unix-domain-socket server at `/run/willbuy/broker.sock`. Single-shot typed request/response per capture; validates schema; persists artifacts; enforces byte caps. Socket bind-mounted read-only into each capture container at a fixed path; no other host paths exposed.
- **Visitor worker** — Node + TypeScript. Pulls visit jobs, acquires per-backstory lease, calls `LLMProvider.complete()` with backstory + captured page, validates JSON, runs schema-repair retries (each fresh-context, each a new logical request) and transport retries under a stable logical-request key, writes `visits` and `provider_attempts` rows.
- **Aggregator worker** — Runs inside the pinned scientific-stack container image when all visits reach terminal state or the 3-min timeout from `studies.created_at` fires. Embedding + HDBSCAN clustering, conversion-weighted score, paired statistics (with disagreement handling). Holds single-writer lock on the study before transition to `ready`.
- **Reconciliation job** — Daily. Compares provider billing line items against every `provider_attempts` row (not just visits). Resolves `indeterminate` into refund or commit; writes `cost_discrepancies` on > 1¢ unexplained deltas.
- **LLM adapter layer** — `LLMProvider` (chat) + `EmbeddingProvider`. Capability flags declare idempotency support, zero-retention support, structured-output support, and prompt-caching support (used ONLY on the static prefix per §1). AST-based CI lint forbids the reserved identifiers list at call sites AND in type definitions.
- **Postgres (self-hosted Supabase)** — state transitions use `SELECT … FOR UPDATE SKIP LOCKED` where concurrent.
- **Object storage (Supabase storage, envelope-encrypted)** — 30-day TTL on a11y/transcripts, 7-day on screenshots; account/study delete purges immediately.
- **Stripe** — Checkout for credit packs; signed webhook idempotent on event id.
- **Resend** — transactional email (cap warnings, share-link notifications, account events, account-deletion window notice).
- **Cloudflare** — DNS, SSL, CDN, WAF rate limits; `/r/*` cache bypass page rule; log-field configuration strips query strings on `/r/*`.

### 4.2 Key boundaries
- **Backstory isolation.** One backstory in exactly one in-flight LLM context at a time, enforced by per-backstory lease. Paired A/B is a DB join over results.
- **Fresh-context invariant + static-prefix cache carve-out.** Every LLM call constructs a new message array. Prompt caching is permitted ONLY on a static prefix that contains no variable data (§1); adapter test asserts byte-identical hash across visits.
- **Capture-once-per-URL-per-study.** All N visitors consume the same capture snapshot.
- **Sandbox containment.** Capture workers cannot reach host services except via the Capture Broker Unix socket (single typed one-shot request). Egress allow-listed per-job to resolved IPs; DNS rebinds rejected; IPv4 AND IPv6 private/metadata ranges denied. Container-level boundary (v0.1); kernel boundary via microVM deferred to v0.2.
- **Render boundary.** Captured text + LLM output + cluster labels render as escaped text under strict CSP with a markdown allow-list (§5.10); `dangerouslySetInnerHTML` banned.
- **Pluggable LLM.** Workers depend only on `LLMProvider` / `EmbeddingProvider`.
- **Trust surface around captured content.** Never executed server-side, never used to choose a tool call, only embedded as user-role content. Redaction runs before persist and before submission.
- **Preview isolation.** Per-PR dedicated Supabase + KMS + LLM sub-key; shared sandbox Stripe test-mode and Resend stream with metadata tagging. Fork PRs get stubs only.

## 5. Implementation details

### 5.1 End-to-end data flow — paired A/B (N=30)
1. User submits `{ urls: [A, B], icp, n: 30, seed, screenshots_enabled: false, redact: true, authorization_mode: "verified_domain", acknowledge_third_party_submission: true }`.
2. API server enforces target authorization — both URL eTLD+1 values must be in `account.verified_domains`.
3. Global admission control: if the tenant-wide in-flight budget (§2 #30) is exceeded, return 503 with `Retry-After`.
4. API validates credits, **atomically reserves** estimated cost on the ledger (§5.4) covering the expected visit + embedding + cluster_label calls, creates `Study` + 30 `Backstory` rows (seeded sample).
5. 2 capture jobs enqueued. Capture worker leases a job, starts a fresh hardened container with iptables rules bound to its network namespace before the process is unpaused, runs Playwright under the no-side-effects policy, waits for `networkidle` + lazy-load settle, DOM-removes known banner selectors, extracts a11y tree, (optionally) OCR-scrubs screenshot, redacts BEFORE persistence, sends typed message over the bind-mounted Unix socket to the broker, broker stores artifacts, flips status to `ok` (or `blocked` → study fails, reserve refunded).
6. 60 visit jobs enqueued. Visitor workers pull under per-account (≤ 20) AND global (≤ 100) limits. A visit job acquires the per-backstory lease before starting; if the other variant already holds the lease, the job is requeued with jittered exp backoff (100 ms → 400 ms → 1.6 s capped, unlimited retries until its own visit-job lease expires). Lease-contention retry count is emitted as a metric; > 50 retries on a single job fires a warning.
7. Each visit: atomic spend debit (§5.5) → if reserved, construct fresh-context prompt (system+schema = STATIC cacheable prefix; + backstory; + redacted a11y tree ≤ 30k tokens) → call provider with `Idempotency-Key = logical_request_key = sha256(visit_id||provider||model||'visit'||0)` and `max_tokens=800`. Schema violation → up to 2 schema-repair retries (each a new logical request with `repair_generation` incremented, new logical key, new fresh-context message array, no carried-over assistant turn). Transport error → up to 3 transport retries under the SAME logical-request key. Provider `timeout`/`connection_reset`/`unknown_status` on a non-idempotent provider → classify `indeterminate`, pessimistically debit at 5¢, stop retrying, let reconciliation resolve.
8. When all 60 visits are terminal OR 3-min timeout from `studies.created_at`, aggregator acquires the single-writer lock (§5.11), runs (inside the pinned scientific container image) embedding → HDBSCAN → cluster-labeling LLM calls (all recorded in `provider_attempts`, all spend-checked), paired-δ + paired-t + Wilcoxon + McNemar with disagreement handling. Writes report. Any post-commit late visits land in `late_arrivals`.
9. Ledger `partial_finalize` commits per-ok visit + embedding + cluster_label charges; refunds everything else.

### 5.2 End-to-end data flow — single URL (N default 30)
Same as §5.1 except: 1 capture job; N visit jobs (single variant, no backstory-lease contention); aggregator skips paired-δ, McNemar, and the second variant. Default N=30; min 5; max 100 in v0.1.

### 5.3 State transitions
- `Study`: `created → capturing → visiting → aggregating → ready | failed`.
- `Visit`: `pending → leased → ok | failed`. `failed.terminal_reason` ∈ {`schema`, `transient`, `cap_exceeded`, `provider_error`, `lease_lost`, `indeterminate`}.
- `ProviderAttempt`: `started → ended | indeterminate | indeterminate_refunded`.
- `PageCapture`: `pending → leased → ok | blocked | error`.
- `Job`: `queued → leased → done | requeued | dlq`.
- `BackstoryLease` rows are INSERTed on acquisition, UPDATEd on heartbeat, DELETEd on visit terminal or expiry.
- `AccountDeleteTombstone`: `requested → hot_purged → backup_scrub_complete` (the last transition is a timestamp crossing, not a job).

### 5.4 Credit ledger semantics
- `reserve(account_id, study_id, cents)` — hold at study creation covering all expected provider calls (visits + embeddings + cluster labels). Idempotent on `study_id`.
- `commit(account_id, study_id, provider_attempt_id, cents)` — finalize any completed provider call. Idempotent on `provider_attempt_id`.
- `refund(account_id, study_id, provider_attempt_id, cents, reason)` — release for `failed`/`indeterminate_refunded`.
- `partial_finalize(study_id)` — at study terminal, commit over ended attempts, refund remainder. A study finishes `ready` iff `count(ok visits) ≥ min_ok_visits(n)` where `min_ok_visits(n) = max(5, ceil(0.67 × n))`. So: n=5→5, n=10→7, n=20→14, n=30→20, n=50→34, n=100→67.
- `reconcile_provider_billing(date)` — daily. Pulls provider cost reports, matches per `provider_attempts` row across every call kind. `indeterminate` resolved: provider charge matches → commit; no charge → refund + `indeterminate_refunded`. Unexplained orphan spend → `cost_discrepancies` + operator alert.

### 5.5 Spend-cap enforcement (atomic, race-free, universal)
Every provider-call kind runs this transaction BEFORE the outbound call:
```sql
INSERT INTO llm_spend_daily (account_id, date, cents)
  VALUES ($account, CURRENT_DATE, $est_cents)
  ON CONFLICT (account_id, date)
  DO UPDATE SET cents = llm_spend_daily.cents + EXCLUDED.cents
  WHERE llm_spend_daily.cents + EXCLUDED.cents <= $daily_cap
  RETURNING cents;
```
No row → reservation failed → caller transitions to `failed: cap_exceeded`, refund posted, **no provider call made**. On completion a reconciling row adjusts actual vs estimated. `est_cents` per kind: visit 5¢, embedding 0.2¢, cluster_label 3¢, probe 0¢. The 50%-of-cap email is gated by a `UNIQUE(account_id, date, kind='cap_50_warning')` row in a side table, so exactly one email fires per account per day. §6 includes a 40-concurrent-writer test at 99.9% of cap across all kinds.

### 5.6 Cost model (v0.1 defaults)
- Chat: **`claude-haiku-4-5`** (Anthropic). Price snapshot: $1 per million input tokens, $5 per million output tokens = **0.1¢ per 1k input tokens, 0.5¢ per 1k output tokens**.
- Embedding: **`voyage-3-lite`** (Voyage AI). Price snapshot: $0.02 per million tokens = **0.002¢ per 1k tokens**.
- Visitor call, uncached: ~6k input + ~600 output → `6 × 0.1¢ + 0.6 × 0.5¢ = 0.6¢ + 0.3¢ = 0.9¢`. With prompt caching on the stable system+schema prefix (static per §1 carve-out), the cached portion (~2k input) reads at provider cache-read rate and drops effective cost to ~0.5¢.
- Embedding call: ~30 findings × ~30 tokens = ~900 tokens → ~0.002¢/visit. Per study of 30 visits: ~0.05¢.
- Cluster-label call: ~12 calls/study × ~300 tokens → ~1.5¢/study.
- **Per-visit hard ceiling: 5¢.** Caps all retries and `indeterminate` pessimism.
- Reference run (User story #2): 30 visits × 3.5¢ conservative budget + 1.5¢ ≈ $1.06; two runs ≈ $2.10.
- Credit-pack tiers: $29 (1,000 credits ≈ 285 visits), $99 (4,000 credits ≈ 1,140 visits), $299 (15,000 credits ≈ 4,280 visits). Pricing locked for v0.1; revisited after the benchmark.

### 5.7 Algorithms
- **Backstory sampling.** Each archetype defines categorical distributions plus a free-text quirk draw. RNG seeded by `(seed, study_id, idx)`.
- **Paired-δ.** For each backstory i with both visits `ok`: `δ_i = will_to_buy_B_i − will_to_buy_A_i`. Report mean, 95% CI, paired-t, Wilcoxon signed-rank. Binary `next_action` collapsed per §2 #18 → McNemar's on 2×2 discordant pairs. Disagreement rule per §2 #19.
- **Conversion-weighted score.** Per variant `score = Σ w[next_action_i] / n_ok`.
- **Clustering (deterministic within pinned image).** Lowercase + whitespace-normalize + dedupe + lex-sort findings; embed (L2-normalized); HDBSCAN(`min_cluster_size=3`, `min_samples=3`, `cluster_selection_method='eom'`, `random_state=42`, `approx_min_span_tree=False`); tie-breaks by sorted input order. Runs inside the pinned scientific container image; determinism guaranteed for identical image digest only.
- **Truncation.** A11y-tree ordered: pricing → above-fold CTAs → remaining reading order. Greedy truncate to 30k tokens per the pinned chat-provider tokenizer.

### 5.8 Security / abuse
- API key = 32-byte random, `sk_live_` prefix, hashed in DB. ≤ 2 active per account.
- Captures only in server-side hardened containers (v0.1; microVM in v0.2) with Unix-domain-socket-only host channel and default-deny egress. URL validator blocks IPv4 and IPv6 private/metadata/loopback/link-local/ULA; snapshots resolved IPs and rejects DNS rebinds. Chromium sandbox layers (site isolation, renderer seccomp-bpf, out-of-process iframes) KEPT ENABLED — `--no-sandbox` banned by a CI lint.
- Captured text is untrusted data — user-role content only, never tool-call selection.
- Report URLs use stable non-secret slug + query-string token on first access + cookie-scoped session after redirect. Default `noindex`. Owner can revoke; expired/revoked tokens → 404. Every token-bearing response has `Cache-Control: no-store`, `Referrer-Policy: no-referrer`, CSP. On revoke the CDN cache is purged for the slug.
- Stripe webhook signature verified; `event.id` as ledger idempotency key.
- Render-bomb defenses: per-job CPU/RAM/wall/byte/DOM/host caps; per-account + global concurrency budgets; per-domain and per-provider circuit breakers.
- Browser-stack patch SLO per §5.16.

### 5.9 Data classification & redaction; selector list ownership
All captured content classified `customer-third-party`, never reused outside the requesting account. Redaction runs **before persistence** to object storage and **before LLM submission** per §2 #31. Labeled-context rule for generic 32+ char base64/hex tokens prevents false-positives on asset hashes, nonces, SHAs, SKUs. Second-pass scrub runs on everything that reaches `/r/*`. The redactor is versioned (`redactor_v=1`) and has a false-positive + boundary golden-fixture suite (§6.1).

**Curated selector list.** The cookie-banner and fullscreen-overlay DOM-removal selector lists live at `configs/banner-selectors.yaml` (cookie banners) and `configs/overlay-selectors.yaml` (fullscreen overlays), both owned by the browser-automation engineer (listed in `CODEOWNERS`). Every PR modifying either file requires one additional engineer review AND at least one accompanying capture-fixture regression test demonstrating the new selector is needed and does not remove primary content. CI runs the regression suite on every PR touching these files.

#### 5.9.1 Operational definitions for blocked-classifier signals
- `primary_content_absent := DOM text-node total character count < 200 AND a11y-tree subtree rooted at any node with `role=main` has < 5 descendant nodes`.
- `primary_cta_absent := a11y-tree has ZERO nodes with `role ∈ {button, link}` whose accessible name matches the CTA-verb regex `(?i)\b(buy|start|try|sign[ -]?up|get[ -]?started|add to cart|checkout|contact sales|book|subscribe|download)\b`.

### 5.10 Untrusted-content render boundary
All captured page text, LLM output strings, cluster labels, and URLs displayed on dashboard or `/r/*` use:
- **React auto-escaping** everywhere; `dangerouslySetInnerHTML` banned by ESLint `react/no-danger` (CI-enforced).
- **Markdown** of LLM strings via `react-markdown` with allow-list `[p, ul, ol, li, strong, em, code]`; raw HTML rejected; non-`http(s)` autolinks rejected.
- **CSP** on `/dashboard/*` and `/r/*`: `default-src 'self'; script-src 'self'; object-src 'none'; base-uri 'self'; frame-ancestors 'none'; form-action 'self'; require-trusted-types-for 'script'`.
- **Additional headers:** `X-Content-Type-Options: nosniff`, `Referrer-Policy: no-referrer`, `Permissions-Policy: camera=(), microphone=(), geolocation=(), clipboard-read=(), clipboard-write=()`.
- Captured URLs are rendered as `<code>` text, never as hyperlinks.

### 5.11 Study finalization (single-writer)
The aggregator runs inside a transaction:
```sql
SELECT 1 FROM studies WHERE id = $1 AND status = 'aggregating' FOR UPDATE SKIP LOCKED;
```
Zero rows → another aggregator already holds the lock (or the study already committed to `'ready'`/`'failed'`); the worker logs `aggregator_lost_race` and exits without running HDBSCAN / cluster-label LLM calls / paired-δ. `SKIP LOCKED` matches the capture-queue pattern in §4.1 and avoids blocking a connection for up to the 3-min aggregate timeout while a competing worker finishes — `NOWAIT` was considered and rejected for noisier log semantics (raises `lock_not_available` rather than returning zero rows). Correctness of "exactly one report" does NOT depend on this lock — it is guaranteed by `UNIQUE(reports.study_id)`; the lock exists to avoid duplicate LLM cluster-label spend and wasted compute.

On row acquired: writes `status = 'ready'` only if all visits are terminal and `count(ok) ≥ min_ok_visits(n)`; otherwise `status = 'failed'`. Late-arriving visits that find `status ∈ {'ready','failed'}` write to `late_arrivals` and do NOT mutate the report. The 3-min aggregate timeout is measured from `studies.created_at`; a visit still holding its 10-min lease past that point is allowed to complete but routes to `late_arrivals`. The per-backstory lease is released on visit terminal commit OR `lease_until` expiry.

### 5.12 Field-level logging & observability policy
Structured logs, traces, and error events emit only: `account_id`, `study_id`, `visit_id`, `provider_attempt_id`, `transport_attempt_id`, event name, duration, error class. They NEVER emit: raw URLs, share-token values, API keys, provider payloads, a11y-tree content, LLM output strings, backstory text, email addresses, captured page bytes. URLs are logged as salted SHA-256 (`url_hash`). Share-token values never reach logs because the token arrives on the URL QUERY STRING (stripped by Cloudflare `/r/*` log field config) and is immediately swapped for a path-scoped cookie on 302 — a second application middleware asserts tokens never appear in any structured-log outbound event. API keys are masked to the last 4 chars. Payload-bearing error context goes to the encrypted `error_events` table (30-day TTL, same KMS root) and is referenced from logs by `error_event_id`. Cloudflare log-field configuration strips query strings on `/r/*` (primary control) AND on `/api/*` (defense in depth). Resend message-stream logs exclude email body content (only message id + recipient domain). Lease-contention retry counts, global-in-flight depth, per-provider token-bucket fill level, and circuit-breaker state are exposed as metrics with alert thresholds (contention > 50/job → warn; global depth > 0.9 × cap → warn; circuit breaker trip → page).

### 5.13 Capture Broker design
**v0.1 (container transport):** each capture container runs in its own network namespace with iptables rules bound *before* the container process is unpaused:
- DROP all egress except to resolved target-origin IPs + explicit subresource IPs (up to 50 hosts, IPv4 AND IPv6).
- DROP all inbound except ESTABLISHED/RELATED.
- DROP all traffic to RFC1918, 100.64.0.0/10, 127.0.0.0/8, 169.254.0.0/16, ::1/128, fc00::/7, fe80::/10, 2001:db8::/32.
- No route to host services.

The container reaches the host ONLY via a Unix domain socket at `/run/willbuy/broker.sock`, bind-mounted READ-ONLY into the container at a fixed in-container path. No other host paths are mounted. The container runs rootless with a dedicated UID that owns no host resources; the broker socket is mode `0660` and owned by a group the container UID is a member of. One-shot typed request schema at capture completion:
```ts
{
  status: 'ok' | 'blocked' | 'error',
  a11y_tree_b64: string,        // ≤ 10 MB after decode
  screenshot_b64?: string,      // only when study.screenshots_enabled AND after OCR scrub
  banner_selectors_matched: string[],
  overlays_unknown_present: boolean,
  blocked_reason?: string,
  host_count: number,
  breach_reason?: string
}
```
The broker validates shape + byte caps, writes to object storage, inserts `page_captures` row, returns an ack. No reverse channel for dynamic network requests; DNS resolution happens IN the container but is constrained by the iptables IP allowlist (rebinds rejected at fetch time). Broker enforces its own byte caps as defense-in-depth over the in-container caps. Chromium's own sandbox layers remain enabled (site isolation, renderer seccomp-bpf, out-of-process iframes); `--no-sandbox` is banned by a CI lint.

**v0.2 (microVM upgrade):** the transport layer changes from Unix domain socket to vsock (cid=host, port=9000), and the iptables rules move from the container network namespace to the microVM's tap device. Request/response schema, broker validation, byte caps, and retention stay identical — the broker protocol is transport-agnostic by design, so the v0.2 migration is a host-side + base-image swap with no schema churn.

### 5.14 Global backpressure & rate limiting
- Per-provider token-bucket rate limiter bounded at 0.8× the provider's published rate; shared across all workers via a Redis-like Postgres-backed `rate_tokens` table with SKIP LOCKED + conditional update.
- Global in-flight concurrency budgets per §2 #30; enforced by a `global_inflight` counter table with conditional increment.
- Per-provider circuit breaker: 5 consecutive 5xx in 60 s → open for 60 s; half-open probes single request; closed on success. State is a row in `provider_circuit_state`, read on every call.
- Study-create admission control: when global visit-worker queue depth > 2× global cap, `POST /studies` returns 503 with `Retry-After: 30`. Admin override bypasses admission control but never bypasses the cap.
- All four controls emit metrics; thresholds wired to PagerDuty.

### 5.15 Logical-request idempotency vs transport attempts
- `logical_request_key = sha256(visit_id || provider || model || request_kind || repair_generation)`. For a non-visit provider call (embedding/cluster_label) `visit_id` is replaced by the study-scoped object id (`embedding_batch_id` or `cluster_label_id`).
- All transport retries for the same logical request carry the SAME `Idempotency-Key`. Providers that support idempotency deduplicate; providers that don't → no transport retries on `timeout|connection_reset|unknown_status`, attempt marked `indeterminate`, pessimistic debit, reconciliation resolves.
- Schema-repair retries increment `repair_generation` → NEW logical key. Semantically a repair is "ask again with prior bad output as input," not "retry the same call."
- `transport_attempt_id` is observability only; each transport attempt writes a `provider_transport_log` row (hot-retained 14 days) with `{logical_request_key, transport_attempt_id, status, provider_request_id?, started_at, ended_at, error_class?}`.

### 5.16 Browser-stack patch/update policy
- **Weekly cadence.** Every Tuesday: rebuild capture-container base image with latest stable Chromium + Playwright + host-kernel (container runtime also bumped if a stable release exists). Run full CI regression + benchmark inside the image. Canary at 5% of capture traffic for 24 h; promote to 100% on success; rollback on any regression. (v0.2: cadence extends to cover the microVM guest kernel and Firecracker itself.)
- **Emergency SLO.** Any upstream CVE with CVSS ≥ 7.0 affecting Chromium / V8 / Playwright / container runtime / host kernel: deploy to production within 72 h of upstream patch availability. Detection: subscribed CVE feeds for each component; pager rotation routes alerts.
- **Image provenance.** All capture-container images signed with `cosign`; SHA pinned in deploy manifest; image swap requires two approvals in the deploy pipeline.
- **Rollback.** Previous image retained as `rollback` tag for 30 days; single-command rollback with no data migration.
- **Drill.** Quarterly simulated emergency-patch drill; success criterion is deploy-to-prod within 72 h of simulated CVE disclosure (see §6.3).

### 5.17 Screenshot handling (opt-in)
- Default OFF per study (`screenshots_enabled=false`). Enabling requires an explicit UI toggle at study creation that briefly explains the extra PII risk.
- When enabled: Tesseract OCR runs on the raw screenshot inside the capture container before the Unix-socket handoff; OCR text is fed through the §2 #31 redactor; for every redaction hit, the reporting bounding box is Gaussian-blurred (σ=14, radius=30 px); the blurred image is what gets base64-sent to the broker. Raw pixels never leave the container.
- 7-day TTL. Never sent to LLM providers. Never exposed on `/r/*`. Available to the owning account in the debug view only. Account-delete scrubs immediately.

### 6.1 Red/green TDD — built test-first
- `computePairedDelta` — fixtures for mean, 95% CI, paired-t, Wilcoxon, McNemar (per §2 #18 binarization); **disagreement fixture** (paired-t p<0.05 AND Wilcoxon p≥0.05 → report flag set, conservative p used).
- `computeConversionScore` — weight-map tables; edges (all-leave, all-purchase, single visitor).
- `sampleBackstory` — same seed + idx → identical; marginals match ICP spec over N=10k within tolerance.
- `validateVisitOrRetry` — malformed outputs at every required field; over-cap strings; schema-vs-transient retry classification; retry-budget boundaries; repair retry asserts NEW logical_request_key AND new message array with no assistant-role turn.
- Credit ledger — double-spend impossible, negative balance impossible, concurrent debits under FOR UPDATE, `partial_finalize` fixtures across visit+embedding+cluster_label mixes (§16 unified ledger).
- `atomicSpendReservation` — 40 concurrent callers at 99.9% of cap across ALL provider-call kinds (visit, embedding, cluster_label); cap NEVER exceeded; reservations that fail issue zero provider calls.
- `logicalRequestKey` — transport retries of the same logical request produce an IDENTICAL key; schema-repair increments `repair_generation` and yields a DIFFERENT key.
- `providerAttemptLedger` — every call kind writes a `provider_attempts` row; reconciliation finds all rows; cap debit runs before the outbound call for all kinds.
- `minOkVisitsFormula` — `min_ok_visits(n)` for n ∈ {5, 10, 20, 30, 50, 100}.

### 6.2 CI tests
- **Unit** — all modules above plus a11y-tree serializer, truncator (pinned tokenizer), cluster-labeler adapter, cost estimator.
- **Integration** — full URL→report loop against a fixture server with stub LLM. Target: < 30 s at N=5.
- **Paired-A/B isolation** — (a) a backstory never appears in the OTHER variant's recorded transcript; (b) adapter call logs show distinct logical_request_keys per call across different visits and no continuation identifiers; (c) prompt-assembly call counter increments once per call; (d) per-backstory lease blocks simultaneous A/B under 50-thread contention.
- **Schema-repair freshness** — on repair retry, the request contains zero assistant-role messages AND a distinct logical_request_key.
- **Prompt-caching carve-out** — adapter test hashes the content covered by `cache_control` across a suite of visits that differ only in backstory/page and asserts the hash is byte-identical; additional test asserts that adding a backstory byte into the cached region fails the test.
- **Atomic-spend concurrency** — 40 parallel debits at 99.9% cap across the three provider-call kinds; cap never exceeded; exactly-once 50% email.
- **Resume-after-restart** — kill workers mid-flight at FOUR distinct points (pre-lease; post-lease pre-provider-call; mid-provider-call; post-commit pre-heartbeat); restart; no duplicate visits (UNIQUE + lease); lease reclaim past `lease_until`; logical-request idempotency prevents double-charge for point 3.
- **Reproducibility (pure layers)** — same seed → identical backstories; given fixtured LLM outputs, aggregator produces identical clusters (inside the pinned container image digest) and identical paired-δ. Cross-image reproducibility is NOT asserted.
- **Semantic stability (nightly, with teeth)** — real-provider runs nightly at `temperature=0`: cluster-label string ≥ 80% match vs golden baseline; per-backstory δ within ±0.5. Failure posts a PagerDuty page AND a Slack alert; 3 consecutive nightly failures automatically BLOCK the next production deploy (deploy pipeline reads the `semantic_stability_state` record and refuses to promote on `failed_streak ≥ 3`). Deploy block is overridable by a named operator with a written reason recorded in the incident ticket.
- **Cap enforcement** — seeded at 99% cap; next visit `cap_exceeded` and refunded without provider call; applied to each of the three call kinds.
- **Stripe webhook** — idempotent on event-id under retry storms.
- **Sandbox containment** — capture attempts to reach `169.254.169.254`, `127.0.0.1`, RFC1918 IPv4, `::1`, `fe80::`, `fc00::`, IPv6-reachable cloud metadata aliases, internal hostnames — all fail; DNS-rebind defeated.

### 6.3 Manual / bench tests (ship gates)

## 9. Risks & mitigations (v0.1)
- **Plausible-slop risk.** 3 labelers + separate 4th adjudicator + pre-registered matcher rubric including pre-registered false-positive rubric as hard ship gate; semantic-stability nightly with 3-fail deploy block.
- **Chromium / kernel exploit from capture (v0.1 container posture).** Mitigations: rootless Docker with user namespace + seccomp + AppArmor + read-only rootfs + no-new-privileges + dropped ALL capabilities; Chromium's own sandbox left enabled (site isolation, seccomp-bpf renderer, out-of-process iframes); Unix-socket-only host channel via Capture Broker; default-deny IPv4+IPv6 egress in a dedicated network namespace; ≤ 50 hosts; per-request DNS pinning; no-side-effects browser policy; weekly patch SLO + 72 h CVE SLO; container-hardening CI assertions + `--no-sandbox` grep-ban. **Residual: v0.1 does NOT have a kernel boundary — a Chromium-sandbox-escape chained with a host-kernel exploit pops the host.** This is accepted for MVP targeting indie hackers at $29/mo and is the explicit gating reason for the v0.2 microVM upgrade before SOC 2 / fintech buyer expansion. Residual is documented in the security reviewer sign-off with the v0.2 remediation date committed.
- **Firecracker/vsock red-team (v0.2 upgrade).** Hardened microVM profile; vsock-only host channel via Capture Broker with identical schema; red-team re-run as gate for v0.2 cutover.
- **Browser-RCE ongoing exposure.** Weekly patch cadence + 72-hour CVE SLO + cosign-signed images + rollback tag + patch-SLO drill as ship gate. Treated as continuous operational risk, not one-time.
- **Residual side-effects from permitted GETs.** Analytics/telemetry host denylist + side-effect-path regex + query-token filter. `/r/*` carries a caveats disclosure that 100% blocking is not claimed.
- **Public-report data leak.** Stable non-secret slug + query-string token on first access + scoped cookie after redirect + Cache-Control: no-store + Referrer-Policy: no-referrer + CF query-strip on `/r/*` AND `/api/*` + cookie scoped to exactly `/r/<slug>` + revoke-purges-cache; raw transcripts/screenshots never on public surface; 30-day a11y TTL, 7-day screenshot TTL.
- **Screenshot PII.** Opt-in default-off + Tesseract OCR scrub before persistence + 7-day TTL + never to LLM/`/r/*`.
- **LLM cost blowouts.** Unified `provider_attempts` ledger across visit+embedding+cluster_label + atomic conditional-increment reservation + 50% warning + per-visit 5¢ ceiling + daily reconciliation + pessimistic `indeterminate` accounting.
- **Bot-detection on client pages.** Tightened classifier with operational definitions + CF false-positive fixture suite + graceful refund; stealth bypass v0.2.
- **Provider lock-in.** Adapter interfaces day 1; chat-provider fallback v0.3.
- **Prompt-caching vs fresh-context drift.** Static-prefix-only carve-out + adapter-hash test + listed forbidden continuation identifiers enforced by AST lint.
- **Provider retry ambiguity / duplicate-charge.** Stable logical-request key reused across transport retries; separate transport-attempt records for observability; `indeterminate` state + daily reconciliation.
- **Fleet exhaustion under multi-tenant load.** Global in-flight budgets + per-provider token buckets at 0.8× published rate + per-provider circuit breakers + `POST /studies` admission 503.
- **Self-hosted Supabase ops burden.** DevOps hire + restore + tombstone drill + explicit 21-day deletion window communicated to users. Escape hatch: switch to hosted Supabase if self-host incident rate exceeds budget in Sprint 3. (Self-host is mandated by the project idea; we do not relitigate it.)
- **Statistical overclaim.** paired-t + Wilcoxon + 95% CI + n_ok always shown; explicit disagreement flag and conservative p; McNemar binarization named with weight-vs-binarization reconciliation note; N<20 runs display a low-power warning.
- **Render bombs / asset floods.** Per-job CPU/RAM/wall/byte/DOM/host caps; per-account + global concurrency budgets; per-domain and per-provider circuit breakers.
- **Stranded jobs after partition.** Leases + heartbeats + reclaim; DLQ for operator review; lease-contention metric + alert threshold.
- **Captured PII / secrets reaching providers.** Redact-before-persist and before-submission with labeled-context boundary rule + golden-fixture boundary suite; default-on; explicit consent; zero-retention provider endpoints where available.
- **Stored XSS via captured content.** React auto-escaping; `dangerouslySetInnerHTML` banned; markdown allow-list; strict CSP; captured URLs rendered as `<code>` text.
- **Secret leak via preview envs.** Fork PRs stubs-only; per-PR Supabase + LLM sub-key + KMS; shared Stripe/Resend filtered by `preview_branch` metadata.
- **Late-visit mutation of a finalized report.** Single-writer aggregator lock + `late_arrivals`.
- **Log-as-second-data-store.** Field-level logging policy; URL hashing; query-string token strip by CF on `/r/*` AND `/api/*`; encrypted `error_events`; weekly field-log audit.
- **Determinism overclaim on HDBSCAN.** Claim scoped to the pinned container image digest; cross-image reproducibility not asserted; CI reproducibility test runs inside the same image as prod.
- **Curated-selector-list drift / primary-content removal.** Listed owner + CODEOWNERS + regression-fixture requirement on every change.
- **Backup-deletion overclaim.** Explicit documented 21-day maximum window + tombstone replay + confirmation email worded to match; no claim of backup-media rewrite.
